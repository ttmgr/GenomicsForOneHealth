Agent Skills

A machine-readable, inspectable layer over the pipeline collection. It lets LLM-assisted coding agents (Codex, Claude Code, Cursor, Continue, and similar tools) route a sequencing project to the right documented workflow, validate inputs, build commands only from declared templates, parse outputs, run sanity checks, and write audit logs — without inventing commands, parameters, tools, or databases. We open-source it so the workflows are reusable and reproducible, not locked away.

What this is

A workflow layer built for inspectability

This is not a chatbot demo. It is a workflow-orchestration layer designed for inspectability and reproducibility, and it is useful even with no LLM runtime: the hooks and the eval runner are plain Python you can call directly. Each skill is a YAML spec describing one workflow — its domain, supported inputs, tools (with version pins from the repository environment files), required databases, declared parameters, ordered command templates extracted verbatim from the repository, hook references, outputs, caveats, and failure modes.

The local repository is the source of truth. Every curated command must cite the source file it came from. Current public Oxford Nanopore and EPI2ME references are used only for comparison and completeness; they never override the local workflows. The pack is the agent-facing sibling of the human-facing Pipeline Advisor, which already enforces the same discipline.

The pack is built as a portable harness so the engine is reusable beyond this project: a project-agnostic core (the hooks, schema, instruction contract, memory format, and offline checks), a thin Claude Code adapter, and a project layer holding this repository's tool-specific parsers, routing, and tests. The 13 skills below are the optional project content the harness drives — you can lift the core into another repository and point it at that repository's own workflows.

The 13 skills

One skill per documented workflow

Thirteen workflows across the six One Health domains. Each links to its YAML spec on GitHub.

Environmental Metagenomics

Air Metagenomics

Shotgun metagenomics of low-biomass air/bioaerosol samples: read processing → metaFlye assembly → MetaWRAP binning into MAGs → functional/AMR annotation.

air_metagenomics.yaml

Wetland Health — DNA Shotgun Metagenomics (Track 1)

Microbial community, pathogen, and AMR profiling from wetland water DNA: Dorado SUP → Kraken2 (NCBI nt_core) → nanoMDBG → AMRFinderPlus, with plasmid and virulence screens.

wetland_dna_shotgun_metagenomics.yaml

Wetland Health — AIV RNA Consensus (Track 2)

Whole-genome avian influenza consensus from AIV-positive wetland RNA: per-segment Minimap2 mapping → Clair3 variants → masked BCFtools consensus.

wetland_aiv_rna_consensus.yaml

Wetland Health — Viral Metagenomics (Track 3)

Untargeted RNA virome discovery from environmental cDNA via DIAMOND BLASTx against NCBI NR, classified into viral families.

wetland_viral_metagenomics.yaml

Wetland Health — 12S Vertebrate Metabarcoding (Track 4)

Vertebrate host detection from 12S amplicon eDNA: OBITools4 demultiplexing → Cutadapt → VSEARCH OTUs at 97% → MIDORI2 taxonomy.

wetland_12s_vertebrate_metabarcoding.yaml

eDNA Metabarcoding

Zambia eDNA Metabarcoding

Vertebrate eDNA metabarcoding of Zambian water samples (12S/16S), comparing portable Nanopore against Illumina on a shared VSEARCH pipeline.

zambia_edna_metabarcoding.yaml

Food Safety

Listeria Adaptive Sampling

High-resolution Listeria monocytogenes detection from complex food samples, benchmarking Adaptive Sampling against native runs; three assemblers plus AMR/virulence detection.

listeria_adaptive_sampling.yaml

Clinical Isolates & Plasmid Profiling

AMR Nanopore (clinical isolate assembly)

Single purified clinical isolate assembly and AMR profiling, documented around an MDR Klebsiella pneumoniae KPC study: Guppy → Flye → Minimap2/Racon polishing.

amr_nanopore.yaml

CRE Plasmid Clustering

Plasmid characterization and relatedness for carbapenem-resistant Enterobacterales: Flye assembly → Medaka polishing → MOB-suite typing → Mash distances and Pling DCJ clustering.

cre_plasmid_clustering.yaml

Nanopore AMR Host Association

Methylation-aware clinical metagenomics linking AMR-carrying plasmids and chromosomes to their bacterial hosts via shared methylation motifs (Nanomotif) and MOB-suite.

nanopore_amr_host_association.yaml

Veterinary & Zoonotic Surveillance

Avian Influenza Profiling

AIV subtyping and consensus across the latest RNA and DNA Nanopore chemistries: chemistry-specific basecalling, de novo and reference-based consensus, plus IRMA assembly.

avian_influenza_profiling.yaml

From Feather to Fur

Variant-calling workflow tracking H5N1 transmission from avian to mammalian hosts: Minimap2 → Clair3 → inoculum comparison → consensus FASTA.

from_feather_to_fur.yaml

Viability Assessment

Squiggle4Viability

Deep-learning assessment of microbial viability directly from raw Nanopore signal (POD5), plus a supporting metagenomics arm. Requires PyTorch and a GPU.

squiggle4viability.yaml

Use it with a coding agent

Point your agent at the pack

Clone the repository and point any LLM coding agent at the agent_skills/ directory. The agent reads README.md, then the relevant skill YAML, before acting. It routes the request to a single skill (and says so if none fits), validates the real inputs, databases, and tools with preflight.py, builds commands only from the declared command_templates — never inventing parameters — parses outputs with the declared parsers, runs the validation hooks, and writes a JSON audit log preserving the source files.

The agent-agnostic operating contract lives in agent_skills/core/AGENTS.md; Claude Code specifics are in agent_skills/adapters/claude_code/CLAUDE.md; and the full step-by-step procedure is in agent_skills/project/prompts/use_skill_pack.md. 

# inspect a skill
python -c "from agent_skills.hooks import command_builder as cb; \
import json; print(json.dumps(cb.load_skill_yaml('agent_skills/skills/cre_plasmid_clustering.yaml')['skill']['display_name']))"

# run the offline benchmark suite (no LLM, no bioinformatics tools needed)
python agent_skills/evals/run_benchmarks.py

Caveats

Suggestions, not conclusions

All outputs from this pack are suggestions, not biological, clinical, regulatory, or diagnostic conclusions. Commands are built, not executed: the human remains responsible for running them and interpreting the results.
